# OmniRule Blueprint Skill

This skill is the **Mandatory Red Gate** for all engineering tasks. No code shall be written without a verified blueprint.

## Workflow

### 1. Requirements Restatement
- Clear, concise summary of the task.
- Verification of understanding.

### 2. Architecture Impact
- Which files will be affected?
- Are there new dependencies?
- Does it break existing patterns?

### 3. Step-by-Step Implementation
- Phase 1: Preparation (Research, Cleanup)
- Phase 2: Core Logic (Implementation)
- Phase 3: Verification (Tests, Lint)

### 4. Risk Assessment
- Potential side effects.
- Security considerations.
- Performance impact.

## Success Criteria
- Blueprint is approved by the user.
- Every step is actionable.
- Risks are mitigated or accepted.
